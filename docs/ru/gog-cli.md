# GOG CLI: Автоматизируем Google Workspace без боли

Мы используем Google Workspace не просто как почту или таблицы, а как «внешний мозг» нашего дома. Но управлять им через браузер — это слишком долго. Нам нужно, чтобы скрипты сами отправляли отчеты, писали в таблицы и копировали файлы.

## Проблема Headless-окружения

Когда ты ставишь `gog` на сервер, он по умолчанию пытается найти `gnome-keyring` для хранения токенов. На сервере его нет, а ставить графику ради одной утилиты — это лишнее.

## Как мы решили проблему

Мы отказались от `gnome-keyring` в пользу явного управления секретами:

1.  **Секретный бункер**: Мы создали папку `secrets/` с правами доступа `700`.
2.  **Обход Keychain**: Мы храним JSON-файлы с токенами Google прямо там. `gog` можно явно указать путь к конфигу, чтобы он искал токены в нашей локальной папке.
3.  **Безопасность**: Мы используем скрипт-обертку, который при старте сессии проверяет наличие токена. Если его нет, он запрашивает ввод и сохраняет его в защищенной области, позволяя `systemd` задачам работать автономно.

## Практика: Настройка без графики

### 1. Инициализация (Headless)
Чтобы `gog` не искал `gnome-keyring`, мы передаем путь к секретам прямо в переменной окружения:

```bash
# Указываем путь к нашему "секретному бункеру"
export GOOGLE_APPLICATION_CREDENTIALS="secrets/google_creds.json"

# Аутентификация без попытки лезть в системный Keychain
gog auth --no-keyring --config secrets/config.json
```

### 2. Systemd: Запуск в фоновом режиме
Чтобы рутина (например, ежедневный бэкап) работала автономно, создаем файл `~/.config/systemd/user/routine-backup.service`:

```ini
[Unit]
Description=Daily Backup Routine
After=network-online.target

[Service]
Type=oneshot
# Указываем переменные окружения для GOG CLI
Environment=GOOGLE_APPLICATION_CREDENTIALS=secrets/google_creds.json
ExecStart=scripts/routines/daily_backup.sh

[Install]
WantedBy=default.target
```

## Кейсы использования

*   **Логирование рутин (`sheets`)**: Скрипт `logger.py` использует `gog sheets append`. Это наш главный инструмент аудита: мы видим статус каждой задачи, не открывая таблицу.
*   **Уведомления (`gmail`)**: Если рутина падает, скрипт использует `gog gmail send`, чтобы письмо пришло прямо в `Inbox`.
*   **Резервное копирование (`drive`)**: Доступ к файлам через терминал позволяет делать бэкапы «на лету» без ручного перетаскивания файлов.

*Совет: В 90% случаев ошибки `403 Forbidden` — это неправильные scope (права доступа) при создании API-ключа. Дайте приложению права на конкретные сервисы, и `gog` станет вашим лучшим другом в автоматизации.*
