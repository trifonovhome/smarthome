# Переход на Docs-as-Code: Управление домом через GitHub

Раньше наши планы хранились в локальных файлах заметок. Это работало, пока проект был маленьким. Но когда количество автоматизаций перевалило за десяток, стало сложно отслеживать, что "в работе", а что "в идеях". Мы решили перенести управление проектом "Smart Home" прямо в репозиторий.

## Что мы внедрили

### 1. GitHub Projects (Kanban)
Теперь у нас есть доска с колонками *Idea*, *In Progress*, *Done*. Больше нет вопроса "А что там с бэкапами?", достаточно одного взгляда на доску.

### 2. GitHub Issues (Карточки задач)
Мы перестали делать "ментальные заметки". Любая идея — это Issue. В ней мы обсуждаем детали, сохраняем конфиги и ссылки.

### 3. Pull Requests (Code Review)
Теперь мы не правим конфигурацию "на лету" на сервере. Сначала — ветка, потом изменения, потом проверка (PR), и только потом — деплой.

## Результаты
*   **Прозрачность**: Мы всегда видим, над чем работает ассистент прямо сейчас.
*   **История**: Если что-то сломалось, мы легко находим Issue, где обсуждали эту настройку полгода назад.
*   **Культура**: Мы начали относиться к дому не как к набору скриптов, а как к полноценному продукту.

## Вывод
Управление домом как кодом (Docs-as-Code) — это не про сложность, это про спокойствие. Когда всё задокументировано и трекается в системе контроля версий, ошибок становится меньше, а идей — больше.
